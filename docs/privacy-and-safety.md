# Privacy and Safety

This repo is public, so the default rule is: if it would be awkward, private, or risky to show a recruiter, employer, customer, or stranger, it does not belong here.

## Never commit

- API keys or credentials
- `.env` files with real values
- Private job-search notes
- Private employer/application notes
- Contact lists or personal data
- Raw analytics, CRM, community, or customer data
- Raw scraped data with unclear permissions
- Unreviewed agent outputs that may contain hallucinations
- MCP config containing private paths, tokens, or account IDs

## Safe to commit

- Architecture docs
- Sanitised templates
- Public-safe example briefs
- Prompt templates without secrets
- Diagrams
- Roadmaps
- Evaluation rubrics
- Synthetic demo data

## Public/private operating model

- **GitHub:** clean public proof of capability
- **Obsidian:** private brand/reference knowledge and working notes
- **Google Drive:** private operational data and exports
- **Hermes skills:** reusable procedures, published only when safe
- **Environment variables:** local secrets only
- **Human review:** required before publishing externally

## Approval gates

Human approval is required before agents:

- Publish to public channels
- Send messages as a person or brand
- Change campaign budgets or live settings
- Use private customer/community data in output
- Make claims about competitors, customers, or performance

## AI-specific risks

- Hallucinated claims
- Overconfident strategy based on weak evidence
- Generic copy that sounds plausible but says little
- Accidentally storing personal data from communities or contacts
- Publishing drafts before proper review
- Letting connector write access become too broad

## Safety rule

Agents can draft, summarise, compare, and recommend. A human approves public-facing work and anything involving personal data, employer relationships, customer relationships, budgets, or reputational risk.
